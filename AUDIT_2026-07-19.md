# NYSC Facilities Hub — Full Product Audit (2026-07-19)

Live walkthrough on the dev server (admin + `preview_role` standard/court_aide), desktop + mobile,
dark + light, plus code checks. Console and network were clean on every route visited; `tsc --noEmit` passes.

## What the app is going for (and what it actually is)

The app wants to be **the single operational hub for two courthouse buildings**: rooms, issues,
supplies, keys, lighting, staffing (term sheet), with role-scoped views for admin, court aides,
officers, and standard staff. The identity is coherent — judicial navy chrome, one sidebar, one
header pattern — and the strongest features (Term Sheet, Supplies ordering, My Requests, Help &
Guides) are genuinely production-quality.

The honest read: **it's two apps in one skin.** The *request/fulfillment* app (supplies, issues,
my-requests, work center) is mature, has real data, and works end to end. The *asset-management*
app (lighting, tasks, parts of keys) is scaffolding — full UIs standing on nearly zero data,
which makes the product feel bigger and emptier than it is.

## What's working well

1. **Term Sheet** — clearly the crown jewel. Term selector with real dates, 33 parts, shared-room
   support, directory, PDF/CSV export, notify-everyone, offline pocket copy. Nothing to fix here
   at the audit level.
2. **Help & Guides** — one of the best help pages I've seen in an internal tool. Practical
   quick-start ("set your home room first"), plain-language FAQ, interactive tours.
3. **Make a Request form** — "Where / What / When can it happen?" with a concrete example
   placeholder. Human copy, right level of simplicity.
4. **My Requests** — item summaries as row titles, resolved room names, status chips, request IDs.
   (The old UUID problem is fixed.)
5. **Role scoping** — court-aide nav (Work Center, Tasks, Supply Room, Inventory, Term Sheet) and
   standard nav (Dashboard, My Requests, Term Sheet, Notifications, Profile) are correctly trimmed;
   preview banner works.
6. **Technical health** — zero console errors across every page visited; zero failed network
   requests; typecheck clean. Auth flow (login, 2-step signup with password-strength meter,
   pending-approval, MFA routes) is complete.
7. **Mobile** — real bottom tab bar, pull-to-refresh, install page, FAB for request flows.

## Critical / embarrassing (fix first)

1. **Watermarked Alamy stock photo shipped as the "standard courthouse image"**
   (`public/lovable-uploads/83d441a4….webp`, used by `BuildingCard` as `fallbackImage`).
   The 111 Centre card on the admin dashboard shows visible "alamy" watermark tiles.
   That's an unlicensed-image liability *and* looks broken. Replace with a real photo of
   111 Centre (or a neutral illustration).
2. **"Unknown Space" on the user dashboard issue list** while the same issue shows
   "Room 1727" on the admin dashboard. Room resolution is broken in the user-dashboard
   issues query — exactly the kind of trust-killer that makes staff stop reporting.
3. **Fulfillment dead-ends are visible in the live data**: an order "Ready for Pickup"
   since **Jul 7 (12 days)** in Work Center, and "Chairs — Awaiting approval, 23 days" in
   My Requests. Known open product decision ("Will Deliver" orders have no completion
   action; `completeOrder()` exists uncalled). The queues age silently: nothing escalates,
   nudges, or expires. Decide the terminal action for every status × delivery method and
   add an "aging" surface for both requester and fulfiller.

## Design & consistency (the "design chief" pass)

4. **Every section has 2–3 names.** Nav "Spaces" → header "Room Management" → page title
   "Locations". Nav "Operations" → "Operations Center" → "Facility Operations". "Admin Center"
   → "Supply Requests Management" → "Supply Requests". My Profile → "Profile & Settings".
   Pick one name per section and use it in nav, top bar, and page title. This is the single
   cheapest consistency win in the app.
5. **Two metric-display languages.** Keys/Operations use the slim StatStrip (good, per the
   July look-and-feel pass); Tasks and Work Center still use big legacy stat-card grids —
   Tasks shows four large cards that are all `0`. Standardize on StatStrip.
6. **Counts don't reconcile across pages.** Admin→System says 577 inventory items & 186 spaces;
   Inventory says 252 in catalog; Spaces says 172 rooms + 6 common areas. Keys: 126 physical,
   4 assigned, 118 available — the missing 4 are unexplained. Real users notice this and
   conclude the numbers are fake. Define one source per metric and label the rest
   ("includes inactive…", "2 lost/damaged").
7. **Raw internals leak into the activity feed**: "Changed: parent_room_id",
   "Changed: room_type, description", plus near-duplicate rows for one edit. Map column
   names to labels and coalesce bursts.
8. **Redundant chrome on every page**: a "Back" + "Dashboard" breadcrumb row even on
   top-level pages reached from the sidebar (Home even shows Back on the dashboard itself);
   two notification bells on user dashboard (top bar + in-page, only the in-page one shows
   the 9+ badge); "Signed in successfully. / You've successfully signed in." toast says the
   same thing twice.
9. **Empty-state-first landings.** Keys opens on Lockbox with "Please select or create a
   lockbox"; Supplies auto-selects the alphabetically-first item, which is a full-size scan
   of a plea-bargain form; the supply admin page leads with an email-settings card while the
   actual approval queue sits below the fold. Lead with the work, not configuration or
   empty prompts.
10. **Accessibility**: the header icon buttons (+ New, search trigger, theme toggle) have no
    aria-labels; quick-filter chips overflow and clip ("Wome…"). Minor, cheap to fix.

## Feature practicality — too much, too little

11. **Lighting is a UI without a feature.** Operations→Lighting has three sub-tabs; the Rooms
    table renders 172 rows of "None tracked / Unknown / Unknown", while the dashboard claims
    "Lighting health 100%" off 2 tracked fixtures. Either seed real fixture data and make the
    lighting-issue path primary, or collapse Lighting to a single "report a lighting problem"
    entry until the data exists. Don't show a vanity 100%.
12. **Tasks is over-built for its usage.** 4 stat cards + 6 type chips + 5 priority chips +
    5 status tabs filtering zero tasks. Collapse the filter apparatus until there's volume.
13. **Supply catalog data hygiene undermines a good ordering UX.** Alphabetical sort makes the
    first screen "2240-382, 305A, 305X, 36A…"; toner exists under two naming conventions
    (bare "305A" *and* "Toner 51A"), so the same category is scattered; some items have no
    category ("80X", "C.I.S"); only 6/153 items have pack/unit info so quantity meaning is
    ambiguous on most tiles. A one-hour data pass (rename SKUs to "Toner 305A", categorize
    strays) + default sort by category would transform the first impression.
14. **Room-name data entry is inconsistent** (rooms named after judges vs. functions;
    "Captian Office" typo in both buildings; "Part VACANT"). Consider separating "room name"
    from "current occupant" more strictly — occupants move, room names shouldn't.
15. **"Running security checks…"** — an admin diagnostic RPC runs on every dashboard mount and
    renders its loading line in the middle of the page. Move it to Admin→System; on the
    dashboard show nothing unless there's an actual finding.

### Round 2 (same day): color toners, packaging, remaining UI
- **Color-toner model (live DB)**: HP 305A and 414A/414X split into Set + per-color items.
  Research-confirmed: 305A = CE410A/411A/412A/413A (B/C/Y/M); 305X = CE410X high-yield BLACK only;
  414A = W2020A–W2023A; 414X = W2020X–W2023X (all four colors). Existing rows became the **Set**
  items (kept quantity ≈ sets; flagged for recount); new per-color rows start at 0 for shelf counts
  of loose singles. All in 17 SR Storage. **Toner 410A (qty 29) split the same way**
  (CF410A/411A/412A/413A); "410XU-R" identified as remanufactured high-yield black (CF410X
  equivalent) and labeled accordingly — no color split needed.
- **Packaging notes (live DB)**: 54 commodity items got industry-standard packaging notes
  (envelopes 100/box, pads 12/pack, clips 12/box, paper-clips 100/box, staples 5,000/box,
  markers/highlighters/pens dozen/box, index cards 100/pack, hanging folders 25/box,
  manila folders 100/box, sheet protectors 100/box, rubber bands 1-lb bag, legal paper

- **Aging badges**: admin Supply Requests rows + Work Center panel show "Aging · Nd"/"waiting Nd"
