# NYSC Facilities — Audit & Remediation (2026-06-09)

Follow-up to [AUDIT_2026-06-07.md](AUDIT_2026-06-07.md). This session both audited and **fixed**. Verified on a fresh pull of `main` (0a4059ad) with typecheck, full build, 35/35 unit tests, and a live walkthrough of every major route in the running app.

---

## Fixed this session

### Security (applied as Supabase migrations on project `fmymhtuiqzhupjyopfvi`)

1. **`demote_from_admin` anon-callable (was C1)** — `REVOKE EXECUTE ... FROM anon, PUBLIC`. Migration: `revoke_anon_security_definer_functions`.
2. **`handle_trusted_signup` — genuine loophole.** SECURITY DEFINER with **no caller check**: any anonymous visitor could invoke it for any user id and trigger auto-approval/role assignment. Revoked from `anon`, `authenticated`, and `PUBLIC` (it only needs to run from the email-confirm trigger / service_role).
3. **`on_auth_user_email_confirmed`** trigger function was executable by `PUBLIC` — revoked (triggers don't need EXECUTE at fire time).
4. **Login brute-force rate limiting was silently dead.** `check_rate_limit`, `validate_email_format`, `validate_password_strength`, `sanitize_input` were granted only to `authenticated`, but the client calls them **pre-auth** during login/signup. Every call failed and the client failed open. Granted `EXECUTE ... TO anon` (all are safe: immutable validators + the purpose-built rate limiter). Verified end-to-end: `app_rate_limits.last_attempt_at` now updates on login, console clean. Migration: `fix_preauth_security_rpcs`.
5. **`log_security_event` had two overloads** (`resource_id text` vs `uuid`) — PostgREST rejected *every* call as ambiguous, so security event logging never worked even for admins. Dropped the `text` overload.
6. **Anonymous storage enumeration (was H1).** Six public buckets had `USING (true)`-style SELECT policies (courtroom-photos alone had **four** overlapping ones). Consolidated to one `TO authenticated` policy per bucket. Buckets are `public=true`, so image loading via CDN URLs is unaffected; anonymous listing via the storage API is now blocked. Migration: `restrict_storage_listing_to_authenticated`.
7. **ChartStyle XSS hardening (was C3)** — config keys now validated against `/^[a-zA-Z0-9_-]+$/` before CSS interpolation (colors were already sanitized).

### Schema drift — broken features restored

8. **Lighting page was erroring on load** (`Error fetching open lighting issues`) and **fixture creation was broken**: client code filters/sorts/writes `lighting_fixtures.building_id` and `.emergency_circuit`, but the columns never existed in the DB. Added both (with FK to `buildings`, backfill from `floors`, and an index). Migration: `add_missing_lighting_fixture_columns`. Lighting page now loads clean.

### Real code bugs

9. **`Keys.tsx` conditional hooks (rules-of-hooks)** — `if (isMobile) return` sat *above* `useQuery`/`useDataState`; crossing the mobile breakpoint would crash the page. Moved the early return below the hooks.
10. **App shell width bug** — Layout's root div is a flex item inside shadcn's `SidebarProvider` wrapper but had no `w-full`, so the whole app shrank to content width (~430px of dead space on narrow pages like User Dashboard at 1400px). One-class fix in `Layout.tsx`.
11. **Duplicate breadcrumbs** on 6 pages (TermSheet, SupplyRequests, KeyRequests, SupplyRoom, MySupplyRequests, MyRequests) — pages rendered their own `<Breadcrumb />` on top of the Layout's global one. Removed page-level copies.
12. **`public/sw.js`** — `clients.openWindow` → `self.clients.openWindow` (undefined global).
13. **Useless try/catch wrappers** removed in `excelUtils.ts`; `createSpace.ts` catch now logs before rethrow.
