# NYSC Facilities — Deep Audit (2026-06-07)

Scope: full sweep — security, database/RLS, code quality, performance/bundle, tests/CI. Findings ordered by severity. File refs use `path:line`.

Stack snapshot: React + Vite + TS, Supabase (project `fmymhtuiqzhupjyopfvi`), Tailwind, Tanstack Query, R3F. 863 TS/TSX files (~130K LOC), 58 routes, 21 test files, 4 Deno edge functions, 12MB dist.

## CRITICAL — fix before next deploy

### C1. `public.demote_from_admin` is anon-callable via PostgREST
Supabase Security Advisor `0028_anon_security_definer_function_executable`: the `anon` role can call `POST /rest/v1/rpc/demote_from_admin` and demote any admin. This is the single highest-impact finding.
- **Action:** `REVOKE EXECUTE ON FUNCTION public.demote_from_admin(uuid) FROM anon;` and verify with `\df+`. Also recheck `secure_promote_to_admin`, `promote_to_admin`, `assign_user_role`, `admin_*` for the same.
- Ref: https://supabase.com/docs/guides/database/database-linter?lint=0028_anon_security_definer_function_executable

### C2. ~198 `SECURITY DEFINER` functions executable by `authenticated`
Advisor `0029` — every privileged function in `public` (including `add_admin_user`, `admin_delete_user`, `admin_update_user_role`, `change_user_password`, `delete_room_cascade`, `secure_role_assignment`, `setup_emergency_admin`, etc.) can be called by any logged-in user. Internal `is_admin()` checks inside the function body help, but the surface area is enormous.
- **Action:** for each privileged function: either `REVOKE EXECUTE ... FROM authenticated` (call via a thin RPC that checks `is_admin()` first), or convert to `SECURITY INVOKER` if the elevated context isn't needed. Audit candidates first: `add_admin_user`, `admin_*` (15), `*promote_to_admin*` (3), `*demote*` (2), `create_emergency_admin`, `setup_emergency_admin`, `change_user_password`, `secure_role_assignment`, `assign_user_role`, `remove_user_role`, `ensure_admin_user`.

### C3. CSP allows the app to be served from a remote origin without script-src nonce
[netlify.toml:55](netlify.toml:55) sets `script-src 'self'` but `style-src 'self' 'unsafe-inline'`. Acceptable, but several issues:
- `dangerouslySetInnerHTML` is used at [src/components/ui/chart/styles/ChartStyle.tsx:57](src/components/ui/chart/styles/ChartStyle.tsx:57). The CSS is generated from chart `config` keys/colors. If a user can influence those values (e.g. user-named metric series), it becomes a stored XSS vector. Sanitize keys to `/^[a-z0-9_-]+$/i` and color values to a strict hex/hsl regex before interpolating.
- No `report-uri`/`report-to` configured.

## HIGH

### H1. Six public storage buckets allow object listing
Advisor `0025`: `storage.avatars`, `storage.courtroom-photos`, `storage.inventory-photos`, `storage.issue-photos`, `storage.room-photos`, `storage.term-sheets` have broad SELECT policies. Anyone who guesses the bucket name can enumerate every object via the `storage.objects` listing API.
- **Action:** scope SELECT to authenticated users or owners; remove `USING (true)` policies; for assets that must be browser-loadable, switch to signed URLs.

### H2. HaveIBeenPwned password-leak protection disabled
Advisor `auth_leaked_password_protection`. With users self-registering, enable it: Supabase Dashboard → Auth → Policies → Password Strength → "Check passwords against HaveIBeenPwned".

### H3. RLS perf disaster: 345 `multiple_permissive_policies` warnings across 95 tables
Postgres `OR`-unions every matching permissive policy on every row. Worst offenders (find tally in advisor output): `issue_type_templates` (24), `personnel_profiles` (24), `room_shutdowns` (24), `staff_tasks` (18), `key_requests` (14), `lighting_issues` (14), `form_submissions` (12), `buildings`/`doors`/`floors`/`inventory_categories`/`maintenance_schedules`/`system_config` (9 each). Example pattern: `buildings` has both `"Admin can manage buildings"` (admin-only) and `"Authenticated users can view buildings"` (everyone) on `SELECT` — collapse into one policy with `USING (is_admin() OR true)`.
- **Action:** consolidate; one policy per (role, action). High-traffic tables first.

### H4. RLS perf: 199 `auth_rls_initplan` warnings — re-evaluating `auth.uid()` per row
Wrap with `(select auth.uid())` so Postgres caches the value once per query. Top tables: `profiles` (`profiles_read`, `profiles_self_update`), `key_requests` (9 policies), `supply_request_items` (5), `court_report*` (4 each), `staff_tasks` (4), `scheduled_reports` (4), `room_notes` (4), `occupant_room_assignments` (4).
- **Action:** rewrite policy bodies — measurable latency win on list/dashboard queries.

### H5. 60 unindexed foreign keys
Notable: `key_requests` (`approved_by`, `key_id`, `rejected_by`), `staff_tasks` (2), `key_orders` (2), `floorplans` (2), `maintenance_requests` (2), `backup_restorations` (2), `relocation_schedule_changes` (2), `scheduled_reports` (2), `verification_requests` (2). These cause full table scans on parent deletes and on filter-by-FK queries.
- **Action:** `CREATE INDEX CONCURRENTLY` on each — script-generatable from advisor output.

### H6. 258 files import the Supabase client directly outside `src/services/`
The repo has an ESLint `no-restricted-imports` rule explicitly forbidding this (see error in `src/utils/titleToRoleMapping.ts:9`) — yet 258 source files violate it and the rule has been let through. Plus tests (`tests/e2e/global-setup.ts:2`, `tests/e2e/helpers/auth.ts:11`, `tests/manual-*.spec.ts`) import `@supabase/supabase-js` directly. The architectural guardrail is broken in practice.
- **Action:** either enforce (turn rule into pre-commit + CI fail) and migrate violators, or admit defeat and drop the rule. The middle ground (rule exists, 258 violations) is worse than either.

### M5. Edge functions: CORS falls open without env config
All four functions (`export-rooms`, `import-rooms`, `parse-pdf`, `support-chat`) default to `Access-Control-Allow-Origin: *` when `ALLOWED_ORIGINS` env var is unset, and `originAllowed()` returns `true` for non-browser requests. In production this should be fail-closed.
- **Action:** set `ALLOWED_ORIGINS` for the deployed functions; consider hardcoding the production origin as a fallback.

### M6. Session storage for auth tokens is good — but `detectSessionInUrl` + sessionStorage tradeoff
[src/lib/supabase.ts:8-22](src/lib/supabase.ts:8) — `storage: sessionStorage` is the right call against XSS persistence. Confirm the OAuth/magic-link flow still works end-to-end (sessionStorage is per-tab; the auth redirect must land in the same tab).
