## üöÄ GET STARTED IN 3 STEPS

### STEP 1: Apply Database Migration (5 minutes)

1. Open Supabase Dashboard ‚Üí SQL Editor
2. Copy contents from DAILY_SESSIONS_MIGRATION.txt
3. Paste and run the migration
4. Verify success with:
   ```sql
   SELECT COUNT(*) FROM court_sessions;
   SELECT COUNT(*) FROM coverage_assignments;
   ```

### STEP 2: Access Daily Sessions (1 minute)

1. Navigate to Court Operations Dashboard
2. Click "Daily Sessions" tab (6th tab, calendar icon)
3. You'll see the Daily Sessions interface

### STEP 3: Create Your First Session (2 minutes)

**Option A: Auto-populate from assignments**
```sql
SELECT auto_populate_daily_sessions();
```

**Option B: Manual entry**
1. Select date, period (AM/PM), building (100/111)
2. Click in table cells to edit
3. Enter judge name, part number, status
4. Press Enter or click away to save

## üìã COMMON TASKS
=============================================================================

### View Today's Sessions:
- Date picker defaults to today
- Select AM or PM period
- Choose building (100 or 111)
- Sessions display automatically

### Copy Yesterday's Sessions:
1. Select tomorrow's date
2. Click "Copy Yesterday" button
3. All sessions copied with same details
4. Edit as needed for new day

### Update Session Status:
1. Click on status cell
2. Select from dropdown:
   - CALENDAR (default)
   - HRG (hearing)
   - PC_CONTD (people continued)
   - JD_CONTD (judge continued)
   - BT (bench trial)
   - TR (trial)
   - ADJ (adjourned)
   - DARK (court not in session)
3. Add status detail if needed
4. Auto-saves on blur

### Assign Coverage:
1. Click "Show Coverage" button
2. Click "Add Coverage"
3. Fill in form:
   - Absent staff name and role
   - Covering staff name
   - Optional time range
   - Absence reason
4. Save
5. Coverage badge appears in sessions table

### Set Estimated Finish Date:
1. Click on "Est. Finish" cell
2. Select date from calendar
3. Useful for trials and continued cases
4. Auto-saves

## üîß TROUBLESHOOTING
=============================================================================

### Sessions Not Appearing?
1. Check date/period/building filters
2. Verify database migration applied
3. Check browser console for errors
4. Try refreshing the page

### Can't Save Changes?
1. Check for validation errors
2. Ensure all required fields filled
3. Verify no conflicts exist
4. Check network connection

### Coverage Not Showing?
1. Verify coverage date matches session date
2. Check period (AM/PM) matches
3. Ensure building code matches
4. Refresh coverage panel

### Real-time Updates Not Working?
1. Check Supabase realtime enabled
2. Verify network connection
3. Check browser console
4. Try refreshing page
